あなたはシニアFlutterエンジニアであり、卓越したUI/UXデザイナーです。
今夜、IT企業の代表面接に向けた「動くプロトタイプ」を爆速で作成する必要があります。
以下の仕様を満たす、外部アセット（画像や音源）を一切使わず、単一の `main.dart` ファイルだけで完全に動作するFlutter（Web/モバイル両対応）のモダンなコードを出力してください。

【アプリのコンセプト】
「3週間で習慣を置き換える」支援エンタメアプリ。
やめたい悪習慣を、新しい良い習慣に置き換えるための1画面完結の超シンプルなUX。

【UI/UXの仕様】
1. **画面構成**: 洗練されたダークモード、またはポップで清潔感のあるマテリアル3デザイン。
2. **上部ヘッダー**: 「3週間 新習慣置き換えチャレンジ（現在〇日目 / 継続率75%）」のような進捗インジケーターを視覚的に表示。
3. **中央アクション**: 今日の記録を付ける3つの大きなボタン。
   - 「松（完璧に置き換えた！）」
   - 「竹（意識して行動できた！）」
   - 「梅（つい悪習慣が出たが、気づけた！）」
4. **楽しさと応援演出（最重要）**:
   - ボタンが押されたら、Flutterの標準機能（`AnimatedContainer`、`Opacity`、`ScaleTransition`など）を駆使し、画面に大きな絵文字（🎉、🌟、💪など）やカラフルなエフェクトが跳ねるような、派手で「触っていて楽しい」アニメーションを実行する。
   - ボタンの評価（松・竹・梅）に応じた、人をポジティブにさせる応援メッセージを画面中央にダイナミックに表示する（※特に『梅』を押した時も「気づけただけで偉い！次行こう！」とユーザーを絶対に責めないUXにする）。

【技術的要件】
- **Supabase連携**:
  - `supabase_flutter` パッケージを使用する前提のロジックを組み込んでください。
  - ログイン周りは今回はスキップし、ボタンが押されたら「日付・選択した評価（松竹梅）・応援メッセージ」をSupabaseの `habit_logs` テーブルに `client.from('habit_logs').insert(...)` する関数（非同期）をダミーのURL/AnonKeyの定義と共に実装してください。
- **依存パッケージ**: `supabase_flutter: ^2.0.0` 程度を想定（環境構築でエラーが出にくいよう、極力標準の `material.dart` だけでアニメーションを完結させてください）。
